# Project Context
Build a site-specific paginated scraper for Milktooth that feeds into our core engine. 

# Tech Stack
Python, Requests, BeautifulSoup (bs4), JSON

# Core Requirements
- Target URL: `https://milktooth.com/products/switches?products_table%5Bpage%5D=7'
- Import exact shared utilities from `core.core_engine`: `USER_AGENT`, `clean_text`, `fetch_html`, `initialize_database`, `save_item_data`.
- **Parsing Strategy (JSON-First):** Milktooth is a modern e-commerce site. The script MUST first attempt to extract product data by finding hidden JSON objects inside `<script>` tags (look for `application/json`, `__NEXT_DATA__`, or Shopify objects). Only if the JSON parse fails should it fall back to standard BeautifulSoup HTML DOM parsing.
- **Brand Inference:** Create a robust `infer_brand(name)` function using regex. Milktooth carries enthusiast brands, so include patterns for: HMX, Bsun, Keygeek, Lichicx, Geon, TTC, Everglide, Haimu, Wuque Studio, Outemu, Invyr, Gateron, Cherry, Kailh, JWK, Tecsee, KTT, SP-Star, Aflion, Durock, and Gazzew. Default to "Milktooth" if no match is found.
- Implement a `while` loop for pagination, fetching `?page=X` sequentially.
- Include a `time.sleep(2)` delay between page requests so we do not overload their servers.
- Call `save_item_data` for every valid item, passing `vendor_name="Milktooth"`. Ensure the loop handles missing/sold-out prices gracefully without crashing.
